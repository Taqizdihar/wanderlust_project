Daftar Proses Bisnis Toserba Yogya Ciamis

  1. Penerimaan dan Penanganan Pelanggan

  2. Pengadaan Barang Dagangan

  3. Penataan dan Display Produk

  4. Transaksi Penjualan

  5. Pelayanan Pengembalian Barang

#1 Penerimaan dan Penanganan Pelanggan

Aktivitas:

  • Pelanggan masuk ke toko

  • Security menyambut pelanggan

  • Petugas informasi membantu arah atau pencarian produk

  • Pelanggan mulai berbelanja

Model BPMN As-Is (deskripsi):

  • Start Event: Pelanggan Masuk

  • Task: Security Menyambut

  • Task: Pelanggan Bertanya ke Informasi (opsional)

  • Gateway: Apakah pelanggan menemukan produk?

    • Jika ya → lanjut berbelanja

    • Jika tidak → diarahkan oleh petugas

  • End Event: Pelanggan Mulai Belanja

#2 Pengadaan Barang Dagangan

Aktivitas:

  • Kepala gudang memantau stok

  • Jika stok menipis, membuat permintaan pengadaan

  • Divisi pengadaan menghubungi supplier

  • Supplier mengirim barang

  • Barang diterima dan diperiksa kualitas serta jumlahnya

  • Barang disimpan di gudang

Model BPMN As-Is (deskripsi):

  • Start Event: Stok Barang Menipis

  • Task: Permintaan Pengadaan oleh Gudang

  • Task: Hubungi Supplier

  • Task: Supplier Kirim Barang

  • Task: Pemeriksaan Barang

  • Gateway: Barang Sesuai?

    • Jika tidak → retur ke supplier

    • Jika ya → simpan ke gudang

  • End Event: Stok Terisi

#3 Penataan dan Display Produk

Aktivitas:

  • Staf mengambil barang dari gudang

  • Menyusun produk di rak display

  • Mengecek label harga dan barcode

  • Memastikan kebersihan dan kerapian area display

Model BPMN As-Is (deskripsi):

  • Start Event: Barang Siap Display

  • Task: Ambil Barang dari Gudang

  • Task: Susun di Rak Display

  • Task: Cek Label Harga

  • Task: Cek Kebersihan Area Display

  • End Event: Produk Siap Dijual

#4 Transaksi Penjualan

Aktivitas:

  • Pelanggan membawa barang ke kasir

  • Kasir memindai barang

  • Sistem menghitung total harga

  • Pelanggan melakukan pembayaran

  • Kasir mencetak dan memberikan struk

  • Kasir memasukkan uang ke laci kasir

Model BPMN As-Is (deskripsi):

  • Start Event: Pelanggan ke Kasir

  • Task: Scan Barang

  • Task: Hitung Total

  • Task: Proses Pembayaran

  • Task: Cetak Struk

  • End Event: Transaksi Selesai

#5 Pelayanan Pengembalian Barang

Aktivitas:

  • Pelanggan datang dengan barang yang ingin dikembalikan

  • Petugas layanan menerima dan mengecek kondisi barang

  • Jika sesuai ketentuan, diproses pengembalian uang atau tukar barang

  • Admin mencatat transaksi retur

Model BPMN As-Is (deskripsi):

  • Start Event: Pelanggan Ajukan Retur

  • Task: Pemeriksaan Barang

  • Gateway: Apakah Barang Sesuai Ketentuan?

    • Jika tidak → beri penjelasan

    • Jika ya → proses pengembalian/tukar

  • Task: Catat Transaksi Retur

  • End Event: Retur Selesai

Aktivitas:

  • Tim penerimaan mengecek jumlah dan kondisi baju

  • Start Event: Baju Dikirim oleh Supplier

  • Task: Pemeriksaan Jumlah dan Kualitas

  • Gateway: Barang Sesuai?

    • Jika ya → Simpan di Gudang

  • Start Event: Jadwal Penataan Display

  • Task: Ambil Baju dari Gudang

  • Task: Tata Baju di Display

  • Task: Pasang Label Harga & Barcode

  • Task: Cek Kerapian & Kategori

  • End Event: Area Display Siap untuk Pelanggan

#4 Penjualan Baju

Aktivitas:

  • Pelanggan memilih dan membawa baju ke kasir

  • Kasir memindai barcode

  • Sistem menampilkan harga dan total

  • Pelanggan membayar

  • Kasir memberi struk dan pelanggan membawa baju

  • Start Event: Pelanggan Membawa Baju ke Kasir

  • Task: Scan Barcode Baju

  • Task: Tampilkan dan Hitung Harga

  • Task: Proses Pembayaran

  • Task: Cetak & Beri Struk

  • End Event: Transaksi Penjualan Selesai

#5 Penanganan Pengembalian Baju

Aktivitas:

  • Pelanggan mengajukan pengembalian baju

  • Petugas memeriksa struk dan kondisi barang

  • Jika layak dan sesuai kebijakan, retur diterima

  • Proses pengembalian uang/tukar barang dilakukan

  • Dicatat dalam sistem

Model BPMN As-Is (deskripsi):

  • Start Event: Pelanggan Ajukan Retur

  • Task: Verifikasi Struk & Barang

  • Gateway: Apakah Sesuai Kebijakan Retur?

    • Jika tidak → Penolakan & Penjelasan

    • Jika ya → Proses Pengembalian/Tukar

  • Task: Catat Retur dalam Sistem

  • End Event: Proses Retur Selesai
